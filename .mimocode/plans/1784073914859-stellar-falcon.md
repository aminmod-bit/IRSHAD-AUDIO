# IRSHAD-AUDIO: Исправление всех проблем

## Задача
Привести проект в чистое рабочее состояние: убрать лишнее, настроить PWA, исправить конфиги.

---

## 1. Удалить react-router-dom
**Файл:** `package.json`
- Убрать `"react-router-dom": "^6.30.4"` из `dependencies`
- Выполнить `npm install` чтобы обновить lock-файл

---

## 2. Убрать дублирующий vite.config.js
**Файл:** `vite.config.js` — удалить
- Оставить только `vite.config.ts` (base: '/IRSHAD-AUDIO/')
- Vite приоритетно использует .ts, .js — мёртвый код

---

## 3. Убрать dist/ из git tracking
**Файлы:** `.gitignore`, git index
- Убедиться что `dist/` в `.gitignore`
- Выполнить `git rm -r --cached dist/`
- Это не удалит файлы локально, только из git

---

## 4. Настроить PWA
**Файл:** `vite.config.ts`
- Подключить `vite-plugin-pwa`
- Настроить манифест: name "IRSHAD", short_name "IRSHAD", theme_color, icons
- Включить `registerType: 'autoUpdate'`
- Service worker для кэширования аудио и статики

**Файл:** `public/manifest.json` — создать
- name, short_name, start_url, display, background_color, theme_color
- Иконки (сгенерировать из favicon.svg или создать базовые)

---

## 5. Исправить title в index.html
**Файл:** `index.html`
- Убедиться что title = "IRSHAD" (уже так)

---

## 6. Пересобрать и задеплоить
- `npm run build`
- Запушить на main → GitHub Actions автоматически задеплоит

---

## Файлы для изменения
| Действие | Файл |
|---|---|
| Редактировать | `package.json` |
| Редактировать | `vite.config.ts` |
| Удалить | `vite.config.js` |
| Создать | `public/manifest.json` |
| Редактировать | `.gitignore` (проверить dist/) |

## Верификация
1. `npm run dev` — приложение запускается без ошибок
2. `npm run build` — сборка проходит успешно
3. `npm run preview` — работает в production режиме
4. PWA: в DevTools → Application → Service Worker зарегистрирован
